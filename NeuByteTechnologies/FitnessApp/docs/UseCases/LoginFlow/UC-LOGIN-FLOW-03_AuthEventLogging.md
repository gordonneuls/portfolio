# UC‑LOGIN‑FLOW‑03 — MFA Enrollment Sequence

Use Case ID
UC‑LOGIN‑FLOW‑03
Use Case Name
MFA Enrollment Sequence
Module
Login / Authentication
## Purpose
To define the complete end‑to‑end sequence a user follows when enrolling in Multi‑Factor Authentication (MFA) for the first time. This includes prompting the user to enable MFA, generating a TOTP secret, displaying a QR code, verifying the MFA code, and activating MFA on the user’s account. This flow ensures a secure and user‑friendly MFA onboarding experience.
## Primary Actor
Authenticated User (first‑time MFA setup)
## Stakeholders & Interests
• 	User — wants a simple, guided process to enable MFA for stronger account protection.
• 	System — must securely generate and store MFA secrets.
• 	Security Team — requires MFA enrollment to follow best practices for TOTP and secret handling.
• 	Product Owner — wants a friction‑balanced MFA setup that increases security without overwhelming users.
• 	Analytics Team — needs visibility into MFA enrollment rates and failures.

## Preconditions
• 	User is authenticated.
• 	User does not yet have MFA enabled.
• 	System supports TOTP‑based MFA (Authenticator App).
• 	QR code generation service is available.
• 	MFA secret storage is available.

## Postconditions
Success
• 	MFA is successfully enabled on the user’s account.
• 	User is redirected to the Dashboard with a confirmation message.
Failure
• 	MFA is not enabled.
• 	User remains on the MFA enrollment page or sees an error.
• 	MFA secret may be invalidated depending on failure type.

## Trigger
User logs in and is prompted to enroll in MFA, or user voluntarily selects Enable MFA from account settings.

## Main Success Scenario (Basic Flow — Full MFA Enrollment)
1. 	User logs in or navigates to Account Settings.
(BR‑Login‑17)
2. 	System detects that MFA is not enabled for the user.
(BR‑Login‑17)
3. 	System displays the MFA Enrollment Prompt.
(FS‑LOGIN‑MFA‑01)
4. 	User selects Enable MFA.
5. 	System generates a unique TOTP secret for the user.
(BR‑MFA‑01)
6. 	System generates a QR code containing the TOTP secret.
(BR‑MFA‑02)
7. 	System displays the QR code and manual entry key.
(BR‑MFA‑02)
8. 	User scans the QR code using an Authenticator App (e.g., Google Authenticator, Microsoft Authenticator).
(BR‑MFA‑02)
9. 	User enters the 6‑digit TOTP code generated by their app.
(BR‑MFA‑03)
10. 	System validates the TOTP code using time‑based verification.
(BR‑MFA‑03)
11. 	System activates MFA on the user’s account.
(BR‑MFA‑04)
12. 	System displays a confirmation message:
“MFA has been successfully enabled.”
13. 	System redirects the user to the Dashboard.
(BR‑Login‑07)
14. 	System logs the MFA enrollment event.
(Analytics requirement)

## Alternate Flows
A1 — User Cancels MFA Enrollment
• 	Steps 1–3 occur.
• 	User selects Not Now.
• 	System may:
• 	Allow login without MFA (if optional), or
• 	Require MFA enrollment before proceeding (if mandatory).
• 	System logs the cancellation.
A2 — User Cannot Scan QR Code
• 	Step 8 fails.
• 	User selects Enter Key Manually.
• 	System displays the TOTP secret in text form.
• 	User enters the key into their Authenticator App.
• 	Flow resumes at Step 9.
A3 — User Enters Incorrect MFA Code
• 	Step 10 fails.
• 	System displays an error message.
(BR‑MFA‑05)
• 	User may retry code entry.

## Exception Flows
E1 — QR Code Generation Failure
• 	Step 6 fails.
• 	System displays a fallback message with manual entry key only.
• 	User continues enrollment manually.
E2 — TOTP Secret Storage Error
• 	Step 5 or 11 fails due to backend issue.
• 	System displays a generic error message.
(BR‑Login‑20 fallback)
• 	MFA is not enabled.
E3 — Time Sync Issue
• 	Step 10 fails due to device time drift.
• 	System displays:
“Your device time may be incorrect. Please sync your clock and try again.”

## Non‑Functional Requirements
• 	Security: TOTP secrets must be stored securely and never transmitted after enrollment.
• 	Performance: QR code generation must be near‑instant.
• 	Reliability: TOTP validation must be consistent across time zones and devices.
• 	Accessibility: Enrollment screens must support screen readers and keyboard navigation. (SRS‑A11Y‑01)
• 	Analytics: MFA enrollment attempts, failures, and completions must be logged.

## Flowchart TD

    %% Entry
    A[User Authenticated] --> B[System Detects MFA Not Enabled]

    %% Prompt
    B --> C[Display MFA Enrollment Prompt]
    C --> D{User Enables MFA?}
    D -->|No| D1[Cancel Enrollment<br/>Log Event]

    %% Secret + QR
    D -->|Yes| E[Generate TOTP Secret]
    E --> F[Generate QR Code]
    F --> G[Display QR Code + Manual Key]

    %% User Setup
    G --> H[User Scans QR Code]
    H --> I[User Enters TOTP Code]

    %% Validation
    I --> J{TOTP Valid?}
    J -->|No| J1[Display MFA Error<br/>Retry]

    %% Activation
    J -->|Yes| K[Activate MFA on Account]
    K --> L[Display Success Message]

    %% Redirect
    L --> M[Redirect to Dashboard]

    %% Analytics
    M --> N[Log MFA Enrollment Event]

    %% End
    N --> END[MFA Enrollment Complete]
